# User Provided Context

## Focus Areas for Documentation Analysis

**Primary Objective:** Investigate the codebase to identify outdated and redundant documentation

**Key Investigation Points:**
1. **Code-Documentation Alignment** - Verify existing docs match actual implementation
2. **Outdated Information** - Identify docs that no longer reflect current codebase
3. **Redundant Documentation** - Find overlapping or duplicate documentation
4. **Missing Documentation** - Identify gaps where code lacks documentation
5. **Current vs Archived** - Determine if archive docs should be removed or updated

**Analysis Approach:**
- Deep code analysis to understand actual current implementation
- Compare documented features/architecture with real code structure
- Identify deprecated/removed features still mentioned in docs
- Check for new features lacking documentation
- Validate API endpoints, component structures, and workflows

This context will guide the documentation generation process to focus on accurate, current information rather than potentially outdated existing documentation.
